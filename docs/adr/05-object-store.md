# Object Store

Aplikacja potrzebywała perzystencji obiektów, aby umożliwić serwowanie plików wideo, które są przechowywane w systemie plików bądź innym mechanizmie przechowywania danych. W tym celu powstała potrzeba wyodrębnienia abstrakcji ObjectStore, która umożliwiłaby zarządzanie obiektami w różnych miejscach docelowych (in-memory, filesystem, S3). Zapotrzebowanie na takową abstrakcję było pospolite w innych aplikacjach sieciowych i w Rust istnieje już gotowa biblioteka `object_store`, która implementuje dokładnie to czego potrzebuję. Zamiast tworzyć własną abstrakcję nad systemem plików, zdecydowałem się użyć gotowej biblioteki. Początkowo myślałem o zamknięciu jej pod własną abstrakcją, ale doszedłem do wniosku, że nie ma sensu tego robić, ponieważ implementacja byłaby niemal identyczna do tej, którą oferuje wybrana biblioteka.
